# UMMAYA

**U**nified **M**ulti-**M**inistry **A**gent for **Y**our **A**dministration  
읽으면, **엄마야**.

> **대한민국 국민을 위한 AI 행정 길잡이**  
> 복잡한 공공서비스 절차를 자연어 한 문장으로 시작하고, 필요한 단계와 확인 지점을 차근차근 정리합니다.

```bash
npm install -g ummaya
ummaya
```

또는

```bash
brew install --cask ummaya
ummaya
```

> 학술·연구개발 목적의 프로젝트입니다. Anthropic, LG AI Research, FriendliAI, 대한민국 정부 또는 특정 공공기관과 공식 제휴된 서비스가 아닙니다.

## 엄마야, 이거 뭐부터 해야 해?

어릴 때 우리 엄마는 초능력자 같았습니다.

내가 찾을 땐 없던 체육복,  
분명히 봤는데 사라진 양말,  
내일 가져가야 하는 준비물,  
오늘까지 내야 하는 서류.

나는 아무리 찾아도 못 찾는데, 엄마는 늘 한마디로 끝냈습니다.

> “빨래통 봤어?”  
> “두 번째 서랍 열어봐.”  
> “그거 오늘까지 해야 하는 거 아니야?”  
> “찾아서 나오면 어떡할래?”

그리고 진짜 나왔습니다.

그때는 몰랐습니다. 엄마가 대단했던 건 단순히 물건을 잘 찾는 능력이 아니라, **집안의 모든 흐름을 알고 있었다는 것**이었습니다. 빨래가 어디까지 됐는지, 청소는 어디가 남았는지, 설거지는 얼마나 쌓였는지, 내가 뭘 놓쳤는지, 오늘까지 해야 할 일이 무엇인지까지 알고 있었습니다.

어른이 되면 우리가 못 찾는 것도 달라집니다.

전입신고는 어디서 해야 할까?  
자동차 주소 변경도 따로 해야 할까?  
건강보험, 세금, 지원금, 증명서는 어떻게 확인해야 할까?  
이 일은 어느 기관, 어느 포털, 어떤 서류부터 시작해야 할까?

그래서 만들었습니다.

**UMMAYA.**  
읽으면, **엄마야**.

어릴 때 엄마가 내가 못 찾던 물건을 찾아줬다면, 이제 UMMAYA는 내가 놓치기 쉬운 **공공서비스 절차의 흐름**을 찾아줍니다.

**한마디로 시작하고, 분명한 절차로 정리하고, 결정은 사용자가 합니다.**

## UMMAYA는 무엇인가요?

UMMAYA는 대한민국의 공공서비스 업무를 더 쉽게 이해하고 진행할 수 있도록 돕는 **로컬 대화형 CLI 기반 AI 행정 도우미**입니다.

공공서비스는 보통 하나의 포털에서 끝나지 않습니다. 이사, 출산, 세금, 창업, 전세 계약, 재난 피해처럼 하나의 생활 사건에도 여러 기관, 서류, 기한, 증명서, 납부, 신고, 후속 확인이 함께 얽혀 있습니다.

UMMAYA는 사용자가 필요한 일을 자연어로 말하면 다음과 같은 방식으로 도와줍니다.

- 어떤 기관과 서비스가 관련될 수 있는지 정리합니다.
- 필요한 절차를 단계별 체크리스트로 나눕니다.
- 준비해야 할 서류, 확인해야 할 기한, 수수료, 자격 조건을 안내합니다.
- 본인확인, 개인정보, 결제, 공식 제출처럼 민감한 단계는 사용자가 직접 확인하도록 구분합니다.
- 현재 릴리스에서 직접 처리할 수 없는 단계는 공식 채널로 이어질 수 있도록 안내합니다.

UMMAYA가 지향하는 경험은 단순합니다.

> **“엄마야, 이거 뭐부터 해야 해?”라고 묻듯이 말하면, 복잡한 행정 절차의 다음 길을 보여주는 것.**

## 빠른 시작

### 1. UMMAYA 설치하기

npm으로 설치:

```bash
npm install -g ummaya
ummaya
```

Homebrew로 설치:

```bash
brew install --cask ummaya
ummaya
```

### 2. AI 요청을 위한 로그인

UMMAYA는 LLM 응답에 **K-EXAONE-236B-A23B** 모델을 사용하며, 요청을 보내려면 **FriendliAI API 키**가 필요합니다.

UMMAYA를 실행한 뒤 아래 명령어를 입력합니다.

```text
/login
```

FriendliAI API 키는 세션 범위에서만 사용되며 디스크에 저장되지 않습니다. 공개 릴리스의 CLI 사용자는 정부기관 또는 공공서비스 제공기관의 계정 정보를 입력하도록 요구받지 않습니다.

### 3. 자연어로 물어보기

먼저 포털을 찾고, 기관을 찾고, 서류명을 검색하지 않아도 됩니다. 해야 할 일을 평소 말하듯 입력하세요.

```text
이사했어. 전입신고하고 자동차, 건강보험, 학교 관련 주소도 뭐부터 확인해야 하는지 알려줘.
```

UMMAYA는 필요한 절차를 정리하고, 어떤 단계에서 사용자의 확인이 필요한지 보여주며, 공식 채널을 통해 직접 완료해야 하는 업무가 있다면 그 지점을 설명합니다.

## UMMAYA가 잘하는 일

UMMAYA는 복잡한 행정 업무를 “한 번에 대신 처리하는 마법”이 아니라, **사용자가 놓치기 쉬운 흐름을 찾아주는 행정 내비게이션**에 가깝습니다.

- 여러 기관과 포털에 흩어진 공공서비스 경로를 정리합니다.
- 복잡한 요청을 실행 가능한 단계로 나눕니다.
- 필요한 서류, 조건, 기한, 수수료, 후속 조치를 추정하고 안내합니다.
- 일반 안내와 민감한 작업을 구분합니다.
- 본인확인, 동의, 결제, 공식 신고, 개인정보 제출이 필요한 단계에서는 사용자의 확인을 우선합니다.
- 현재 공개 릴리스에서 직접 완료할 수 없는 업무는 공식 서비스로 넘겨받을 수 있도록 설명합니다.

---

## 신뢰와 안전

공공서비스는 실수의 비용이 큽니다. UMMAYA는 행정 업무를 다룰 때 특히 조심스럽게 설계되었습니다.

- 민감한 작업 전에는 의도한 다음 단계를 설명합니다.
- 본인확인, 동의, 결제, 공식 제출, 개인정보 처리는 높은 신뢰가 필요한 순간으로 분리합니다.
- 현재 릴리스에서 직접 완료할 수 없는 단계는 공식 채널을 통해 처리하도록 안내합니다.
- 법률, 세금, 의료, 재난, 긴급 상황처럼 결과가 중대한 문제는 반드시 관련 공식 기관과 최종 확인해야 합니다.
- UMMAYA는 공식 정부 서비스가 아니며, 공공기관의 법적 판단이나 공식 고지를 대체하지 않습니다.

## 현재 공개 릴리스의 API 현실

UMMAYA는 현재 무엇이 실제로 연결되어 있고, 무엇이 안전하게 시뮬레이션되는지 투명하게 구분합니다.

공개 릴리스에서는 공공서비스 정보 조회와 검색 흐름을 중심으로 동작합니다. 반면 정부24, 마이데이터, 간편인증, 증명서 발급, 납부, 공식 제출, 복지 신청, 공과금 변경처럼 기관 권한이 필요한 보호된 업무는 공식 접근 권한이 없는 한 mock API 또는 안내 흐름으로 표현됩니다.

### 실제 조회 가능한 영역

- 공개 접근이 가능한 공공서비스 검색 및 정보 조회
- 운영자 관리 자격 증명 또는 공개 채널을 통해 호출 가능한 서비스 조회
- 사용자에게 필요한 절차와 공식 서비스 경로 안내

### mock으로 표현되는 보호 업무

- 정부24 등 기관 포털의 보호된 업무
- 마이데이터, 간편인증, 공동·금융인증 흐름
- 증명서 발급, 납부, 환급 계좌 등록, 공식 신고·제출
- 복지 신청, 공공요금 변경, 기관 내부 시스템 연동
- 기타 기관 승인, 보안 심사, 계약, 위임 권한이 필요한 업무

### 왜 mock이 필요한가요?

많은 실제 공공기관 API는 개인 연구자나 독립 프로젝트가 바로 호출할 수 없습니다. 보통 기관 승인, 보안 검토, 운영 계약, 법적 위임 권한, 개인정보 처리 체계가 필요합니다.

UMMAYA의 mock API는 비공식 접근, 인증 우회, 내부 시스템 스크래핑, 기관 승인 주장으로 만들어진 것이 아닙니다. 공개적으로 확인 가능한 서비스 도메인, 행정 절차, 예상 요청·응답 구조, 실제 시민 여정을 바탕으로 구성한 **도메인 기반 시뮬레이션**입니다.

### 공공기관과 파트너에게

UMMAYA는 공공기관, 공공 인프라 운영자, civic-tech 파트너가 함께 시민 여정을 검증할 수 있는 평가 환경을 지향합니다.

기관과 파트너는 UMMAYA를 실행해 시민 관점의 행정 흐름을 체험하고, 정책·절차의 빈틈을 찾고, 안전한 mock 흐름을 공식 live integration으로 전환하는 협업을 검토할 수 있습니다.
