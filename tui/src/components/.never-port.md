## Rule for new component imports

When auditing or mirroring a CC source path under `tui/src/components/`:

1. Check whether the path matches any row above. If yes → reject the port; cite this file in PR review.
2. The CI guard `cc-byte-identical-guard.yml` only locks files that **already exist** in `tui/src/`. Files that are intentionally absent rely on this NEVER-PORT registry as the human-review backstop.
3. Adding a new entry requires (a) audit citation, (b) CORE THESIS justification (1P-Anthropic billing / telemetry / growth / cloud-sync), and (c) Initiative-level approval.
